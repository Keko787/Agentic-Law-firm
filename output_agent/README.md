# Output Agent (Stage 5)

The Output Agent is the final stage of the Agentic Lawyer pipeline. It transforms the raw analysis and evaluation results into a structured, plain-language report suitable for non-lawyers. It produces both a structured JSON report for programmatic use and a compiled Markdown report for human reading.

## What It Does

```
AnalysisResult + EvaluationResult
    |
    v
+-----------------------------------------+
| Executive summary (LLM-generated)       |
|   Plain language, 3-4 paragraphs        |
|   Bottom-line recommendation            |
|   Top concerns + strengths              |
+-----------------------------------------+
    |
    v
+-----------------------------------------+
| Report sections (programmatic):         |
|   1. Risk scorecard                     |
|   2. Clause-by-clause findings          |
|   3. Missing protective clauses         |
|   4. Cross-clause issues                |
|   5. Negotiation playbook               |
|   6. Confidence disclosure              |
+-----------------------------------------+
    |
    v
+-----------------------------------------+
| Dual output:                            |
|   - FinalReport (JSON)                  |
|   - Markdown report (.md)               |
+-----------------------------------------+
```

## Report Sections

### 1. Executive Summary
LLM-generated, targeting a non-lawyer audience (business owners, freelancers). Covers:
- Document type and parties
- Bottom-line recommendation (sign / negotiate / walk away)
- Top 3-5 concerns in plain language
- Strengths of the document
- Overall risk level and pipeline confidence

### 2. Risk Scorecard
- Overall risk level and score (0-100)
- Proceed recommendation
- Findings breakdown (critical, major, moderate, minor counts)
- Number of clauses analyzed and missing clauses

### 3. Clause-by-Clause Findings
Each clause section includes:
- Risk badge and level
- Overall assessment
- Individual findings with severity badges, descriptions, legal basis, and suggested revisions
- Market comparison
- Negotiation strategy

Clauses are sorted by severity — critical/high risk clauses appear first.

### 4. Missing Protective Clauses
Lists clauses that should be present but aren't, with:
- Importance level (critical, major, moderate, minor)
- Description of what the clause would protect against
- Suggested language to add

### 5. Cross-Clause Issues
Issues that span multiple clauses (e.g., "The unlimited indemnification in Section 5 contradicts the $100 liability cap in Section 6").

### 6. Negotiation Playbook
Prioritized list of items to negotiate, ordered by severity. Each item includes:
- The clause and finding title
- Concrete revision language from the Analysis Agent's suggested revisions
- Only critical and major findings with suggested revisions are included

### 7. Confidence and Disclaimers
- Pipeline confidence level from the Evaluator Agent
- Pass rate (percentage of clauses that passed quality checks)
- Score validation status
- Grounding warnings for findings not fully supported by retrieved sources
- Legal disclaimer: "This analysis was generated by an AI system. It is not legal advice."

## Data Models (`models.py`)

| Model | Description |
|-------|-------------|
| `ReportSection` | Heading, content (markdown), priority (sort order) |
| `FinalReport` | Complete deliverable: document info, executive summary, risk score, sections, full markdown, pipeline metadata |

## Usage

```python
from output_agent import OutputAgent

agent = OutputAgent(api_key="sk-ant-...")
report = agent.generate_report(analysis_result, evaluation_result)

# Save both formats
report.save("output/final_report.json")
report.save_markdown("output/final_report.md")

# Access report fields
print(f"Risk: {report.risk_level} ({report.risk_score}/100)")
print(f"Recommendation: {report.proceed_recommendation}")
print(f"Confidence: {report.pipeline_confidence}")
print(f"Findings: {report.total_findings} total, {report.critical_findings} critical")
```

## Output

The Output Agent produces two files:

| File | Format | Purpose |
|------|--------|---------|
| `final_report.json` | Structured JSON | Programmatic consumption, integration with other tools |
| `final_report.md` | Markdown | Human reading, can be converted to PDF |

The Markdown report is a self-contained document with headers, risk badges, and formatted sections that can be shared directly with clients or stakeholders.
